Table dbxref

A unique, global, public, stable identifier. Not necessarily an external reference - can reference data items inside the particular instance being used. Typically a row in a table can be uniquely identified with a primary identifier (called dbxref_id); a table may also have secondary identifiers (in a linking table _dbxref). A dbxref is generally written as : or as ::.

IndexesField NameData TypeDescription
* dbxref_id serial
* accession text DEFAULT ''::text The local part of the identifier. Guaranteed by the db authority to be unique for that db.
ver text
  description text
  db_name text source name, ex. EDAM Ontology A database authority. Typical databases in bioinformatics are FlyBase, GO, UniProt, NCBI, MGI, etc. The authority is generally known by this shortened form, which is unique within the bioinformatics and biomedical realm.
  url text
  props jsonb DEFAULT '{}'::jsonb
Indexes
dbxref_pkey ON dbxref_id
unique_dbxref_accession_version ON accession, ver
idx_dbxref_accession ON accession
idx_dbxref_ver ON ver
